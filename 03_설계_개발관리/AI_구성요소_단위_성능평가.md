# AI 구성요소 단위 성능평가 — 디지털의료제품법 제40조 대응

**버전**: v0.1
**작성일**: 2026-04-24
**관련 근거**: 디지털의료제품법(법률 제20722호, 2025-01-24 공포 / 2026-01-24 시행), 동법 시행규칙(2025-02-28 현행), MFDS 2026 업무보고(2025-12-16)

---

## 1. 제도 개요

### 1.1 법적 근거
- 디지털의료제품법 제40조 — 디지털의료제품의 센서·인공지능 알고리즘 등 구성요소에 대한 성능평가
- 시행규칙 — 성능인증 신청·성능검사·성능인증서 발급 절차
- 2026년 MFDS 업무보고 — 자율성능인증제 도입 시행규칙 개정 '26년 1월 예정

### 1.2 목적
완제품 허가 이전 단계에서 센서·AI 알고리즘 등 개별 구성요소의 성능을 선평가·인증하여
- 개발 단계 리드타임 단축
- 동일 구성요소 재사용 시 중복 검증 회피
- K-디지털의료제품의 신뢰성 확보 및 글로벌 진출 기반 조성

### 1.3 적용 대상
- 센서(영상·생체신호·환경)
- AI 알고리즘(진단·분류·검출·예측·생성형 모델 등)
- 기타 완제품에 공통 탑재되는 SW 구성요소

---

## 2. X-ray Detector / SW 프로젝트 영향 분석

### 2.1 해당 구성요소 식별
| 구성요소 | 유형 | 사전 성능평가 적용 후보 |
|---|---|---|
| X-ray detector 센서 (FPD) | 센서 | ◎ (DQE·MTF·NPS 규격 평가) |
| 선량/노출 제어 AI | AI 알고리즘 | ◎ (진단 선량 최적화) |
| 이상 검출/분류 AI (선택) | AI 알고리즘 | ◎ (SaMD 단독 경로) |
| 이미지 후처리 (에지·노이즈) | SW | △ (전통 SW는 대상 외 가능) |

### 2.2 활용 전략
1. **Detector 센서 성능인증**: IEC 62220-1-1 (DQE) + IEC 62220-1-3 (MTF) 기반 규격 확보 → 완제품 심사 시 재활용
2. **AI 알고리즘 인증**: 임상 데이터셋·외부 검증셋 분리 + 공정성·일반화 성능 평가
3. **변경관리**: 인증 구성요소 교체/업데이트 시 PCCP(예상변경관리계획) 또는 재인증 결정 트리 필요

## 3. 성능평가 요구사항 (잠정)

### 3.1 센서 구성요소
- 환경 조건별 성능 재현성 (온도·습도·전원 변동)
- 노화·내구성 데이터
- 제조편차 상한 검증
- 교정·점검 주기 근거

### 3.2 AI 알고리즘 구성요소
- 학습·검증·시험 데이터셋 분리 증빙
- 외부 데이터(다기관·다기종)에서의 일반화 성능
- 공정성(성별·연령·장비 제조사) 검정
- 오류 모드·실패 안전성(Fail-safe) 시나리오
- 입력 분포 드리프트 모니터링 계획(시판 후)
- 생성형 AI 해당 시 추가: 할루시네이션률·출력 안정성(2025-01 MFDS 생성형 AI 가이드라인 준용)

### 3.3 문서 증빙
- 성능평가계획서 (Test Plan)
- 성능평가결과서 (Test Report)
- 데이터셋 기술서 (Data Sheet, Datasheet for Datasets 포맷 권고)
- 모델 기술서 (Model Card 포맷 권고)
- 변경관리 계획 (SOP-CC-001 연계)

---

## 4. 인증 절차 (시행규칙 기반 잠정 플로우)

```
사전상담 → 신청서 제출(계획서·데이터셋·모델 기술서)
        → MFDS 검토 → 성능검사(자체 또는 공인시험기관)
        → 결과 심사 → 성능인증서 발급
        → 유효기간 관리 + 변경 발생 시 재인증/부분 갱신
```

- 유효기간 및 갱신 주기: 시행규칙 최종안 확정 시 재조사
- 인증기관: MFDS 지정 기관(최종안 확인 필요)

## 6. 미확정 사항 (리서치 계속)

1. 시행규칙 최종안의 성능인증서 유효기간

5. 자율성능인증 vs. 의무 평가 구분 경계

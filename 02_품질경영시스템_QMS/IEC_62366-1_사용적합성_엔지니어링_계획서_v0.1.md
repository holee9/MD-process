## 1. 목적
의료용 X-ray 시스템/검출기/소프트웨어의 사용 환경에서 발생할 수 있는 사용오류(Use Error)를
체계적으로 식별·통제하기 위해 IEC 62366-1:2015 + Amd1:2020 의 사용적합성 엔지니어링(UE) 프로세스를
설계·개발 수명주기에 통합 적용한다.

## 2. 규제·표준 근거
- IEC 62366-1:2015 + Amd1:2020 — Application of usability engineering to medical devices
- IEC/TR 62366-2:2016 — Guidance on the application of usability engineering
- FDA "Applying Human Factors and Usability Engineering to Medical Devices" (2016)
- FDA "List of Highest Priority Devices for Human Factors Review" (2022)
- MDCG 2019-16 Rev.1 — Cybersecurity (UI 보안 인터랙션 부분)
- MFDS "의료기기 사용적합성 평가 가이드라인" (2020)
- ISO 14971:2019 §4.1.2 (Use error → Hazard 연계)
- AAMI HE75:2009/(R)2018 — Human Factors Design

## 4. UE 프로세스 단계 (IEC 62366-1 §5)
| 단계 | 산출물 | 주관 |
|------|--------|------|
| 5.1 사용 사양 작성 | Use Specification (의도된 사용·환자집단·환경·사용자 프로파일) | RA + UX + Clinical |
| 5.2 사용자 인터페이스 평가 우선순위 결정 | UI Evaluation Priority List | UX + Risk |
| 5.3 알려진/예측 가능 사용오류 식별 | Known/Foreseeable Use Errors List | UX + Risk + Service |
| 5.4 위험 분석(Hazard-related Use Scenarios) | HRUS Table → ISO 14971 RMF | Risk Manager |
| 5.5 UI 사양 작성 | UI Specification | UX + SW |
| 5.6 UI 평가 계획 | Formative + Summative Plan | UX |
| 5.7 Formative Evaluation | Iterative Test Reports | UX |
| 5.8 Summative Evaluation | Summative Validation Report | UX + 외부 평가기관 |
| 5.9 UE 파일(UEF) 컴파일 | Usability Engineering File | RA |

## 5. Formative vs Summative
| 항목 | Formative | Summative |
|------|-----------|-----------|
| 시점 | 설계 초기/중기 (반복) | 설계 동결 후 1회(주요변경 시 재실시) |
| 참가자 | 5–8명/사용자 그룹 (Nielsen 권장) | 사용자 그룹별 ≥15명, 다중 그룹 시 합계 ≥45명 (FDA 권장) |
| 환경 | 실험실·HiFi 프로토타입 | 실사용 환경(모사 포함) |
| 측정 | 발견적 문제·태스크 흐름 | 위험 기반 Critical Task 통과율 |
| 데이터 | 정성+정량 | 정량(통과/실패) + 근본원인 |
| 합격기준 | 반복 개선 | 사전 정의된 P/F + Use Error → 위험통제 충분성 평가 |

## 6. Critical Task 정의
- 정의: 실패 시 환자/사용자에게 심각한 위해가 발생하거나 진단 정확성에 영향을 미치는 태스크
- 식별 절차: Use Specification → Task Analysis → HRUS → Critical Task 식별
- X-ray 시스템 표준 Critical Task 후보:
  1. 환자정보 매칭(오인 환자 촬영 방지)
  2. 검사 프로토콜 선택(신체부위·연령군)
  3. KV/mAs/AEC 설정 및 Override
  4. 콜리메이션 및 SID 설정
  5. 방사선 노출 발사(Foot Switch)
  6. 후속 노출 차단(Interlock 해제)
  7. AI 진단보조 결과 수용/거부 의사결정
  8. 응급정지(Emergency Stop) 인지·실행

## 7. 사용오류 ↔ 위험관리 연계 (ISO 14971 §4.1.2)
- Use Error는 즉시 Hazardous Situation 발생 가능성 평가 → ISO 14971 위험분석 진입
- 위험통제 우선순위: ① 본질안전설계(Foreseeable misuse 차단) → ② 보호조치(Interlock·Confirm Dialog) → ③ 정보안전(레이블·경고문)
- 위험통제 후 Summative 단계에서 잔존 위험 검증

## 8. AI 인터랙션 추가 요구사항
- AI 진단보조 결과는 EU AI Act Art. 13(투명성)·Art. 14(인간감독) 충족 위해 다음 정보를 동일 화면에 제시:
  - 모델 버전·학습 데이터 출처(학습기간·인구집단)
  - Confidence Interval 또는 위험도 분포
  - 설명성 시각화(Saliency / Heatmap)
  - 사용자 수용/거부 입력 필수
- Critical Task 7번 통과율 측정 시 AI 결과 신뢰편향(Automation Bias) 평가 시나리오 포함
- 합성 이미지(생성형 AI) 출력 시 Art. 50 워터마크 가시성 평가

## 9. 다국가 사용자 그룹 구성
| 시장 | 사용자 그룹 | 요구 인원(Summative) |

언어·UI 로컬라이제이션 검증은 분리 트랙으로 수행하되 본 계획에 포함.

## 10. 평가 환경 및 도구
- HiFi 프로토타입: Workstation 시뮬레이터 + 실제 콘솔 HW
- 모사 환경: 실제 검사실 레이아웃, 시뮬레이션 환자, 시간 압박 시나리오
- 데이터 수집: Eye Tracking(선택), 화면 녹화, Think-Aloud, 태스크 타임 측정, NASA-TLX
- IRB: Summative는 IRB 또는 윤리위원회 승인 필요

## 12. 변경관리 트리거
다음 변경 시 UE 프로세스 재실행 또는 부분 재평가:
- UI 레이아웃·아이콘·인터랙션 패턴 주요 변경
- AI 모델 알고리즘·임계값 변경(SOP-AIGOV-001 §9 연계)
- Critical Task 추가·삭제
- 사용자 프로파일·환경 변경
- 사용오류 시판후 보고(SOP-CVD-001·08_PMS 연계)에서 새로운 패턴 식별

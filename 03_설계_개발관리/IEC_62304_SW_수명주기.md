# IEC 62304 의료기기 소프트웨어 수명주기 프로세스

## 1. 표준 상태
- 현행판: IEC 62304:2006 + Amendment 1:2015 + Amendment 2:2020 (컨솔 통합판 공식 미발행, 3개 문서 조합 적용)
- EN 조화판: EN 62304:2006/A1:2015 (EU MDR 조화표준 OJEU 비등재 → MDCG 2021-5 "state-of-the-art"로 적용)
- FDA 인정표준: FDA Recognized Consensus Standards DB 등재(부분)
- MFDS 적용: 의료기기 소프트웨어 허가·심사 가이드라인에서 참조

## 2. 적용 범위 (프로젝트 관련)
| SW 유형 | 본 프로젝트 적용 예 | IEC 62304 적용 |
|---------|---------------------|----------------|
| 제품 내장 SW (SiMD) | X-ray 시스템 제어 SW, detector 펌웨어, 노출 제어 SW | 전체 적용 |
| 독립 SW (SaMD) | 영상처리·판독 보조 SW | 전체 적용 + IMDRF SaMD 프레임워크 병행 |
| SOUP (Software of Unknown Provenance) | OS, 드라이버, OSS 라이브러리, DICOM 스택 | 식별·검증 요구 (§5.1.5, §8) |
| 비제품 SW (QMS/제조 SW) | MES, QMS 문서관리, 시험장비 SW | IEC 62304 비적용 → 21 CFR 820.70(i) / ISO/TR 80002-2 적용 |

## 3. 소프트웨어 안전성 클래스 (Safety Class)
| 클래스 | 정의 | X-ray 시스템 적용 예 |
|--------|------|----------------------|
| A | 사용자·환자에 상해·건강 손상 가능성 없음 | 로그 뷰어, 유지보수 진단 표시 |
| B | 중대하지 않은 상해 가능 | 영상 후처리 부가 기능 |
| C | 사망·중대 상해 가능 | 노출 제어(kV/mA/타이머), AEC, 방사선 차폐 인터록, 영상 판독 보조(SaMD) |

클래스 결정은 위험통제 SW 기여도 + 고장 시 위해 심각성을 ISO 14971 위험분석 결과에 근거. Amendment 1:2015는 위험통제가 SW 외부 수단으로 완전 이전 가능한 경우 클래스 하향을 허용.

## 4. 수명주기 프로세스 구성
IEC 62304 §5–§9 + 지원 프로세스.

### 4.1 개발 프로세스 (§5)
1. 계획 수립 (§5.1) — SW 개발계획서, SOUP 관리 계획, 통합·검증 전략, 구성관리·문제해결 연계
2. 요구사항 분석 (§5.2) — 기능·인터페이스·성능·사이버보안·위험통제 요구 포함
3. 아키텍처 설계 (§5.3, Class B·C) — SOUP 식별, 인터페이스, 장애 격리
4. 상세 설계 (§5.4, Class C) — 검증 가능 수준 상세화
5. 단위 구현·검증 (§5.5, Class B·C) — 코딩 표준, 단위 시험
6. 통합·시험 (§5.6, Class B·C) — 통합 계획, 회귀시험
7. 시스템 시험 (§5.7) — 요구사항 기반, 결함 회귀시험
8. 릴리스 (§5.8) — 결함 잔존 기록, 저장 매체·무결성, 릴리스 기록 유지

### 4.2 유지보수 프로세스 (§6)
유지보수 계획 수립, 문제·변경 분석, 변경 이행 및 회귀 검증.

### 4.3 위험관리 프로세스 (§7)
ISO 14971 프로세스의 SW 특화 적용 — 기여 SW 항목 식별, 잠재 원인 분석, 위험통제 조치 구현·검증.

### 4.4 구성관리 (§8)
SW 항목 식별, 변경 통제, 구성 상태 기록 (SOUP 포함).

### 4.5 문제해결 (§9)
문제 보고, 분석, 조치, 회귀 확인, 추세 분석.

## 5. Amendment 2:2020 주요 변경
- "Legacy Software" 조항 신설 → 기존 시판 SW를 본 표준 전체 적용 없이도 위험 평가·gap 분석으로 계속 운영 가능
- 사이버보안 요구의 명시적 참조 강화 (IEC 81001-5-1 연계 기반 마련)

## 6. SOUP 관리 실무
- 기능·성능 요구사항 정의, 하드웨어·리소스 요구사항 정의 (§5.3.4)
- 공표 결함(Known Anomalies) 식별 및 위험 평가 (§7.1.3)
- 변경 모니터링 절차 (CVE 모니터링 포함 — IEC 81001-5-1 §6.1 연계)
- OSS 라이선스·공급망 SBOM 관리

## 7. 사이버보안 연계 (IEC 81001-5-1:2021)
IEC 62304 수명주기에 보안 활동을 오버레이: 위협 모델링, 보안 요구사항, 보안 아키텍처 검토, 정적분석·침투시험, 취약점 처리, 패치 관리. FDA 2023 Premarket Cybersecurity Guidance, MDCG 2019-16 Rev.1 요구와 정합.

## 8. AI/ML SW 특화 고려
IEC 62304 기반 수명주기 + IMDRF SaMD N41 + FDA GMLP + AI Act Annex IV. 학습 데이터 거버넌스, 모델 변경관리(PCCP), 성능 모니터링 절차 명문화 필요.

## 10. 실무 체크포인트
- 안전성 클래스 결정 근거를 위험관리 파일에 1:1 링크
- SOUP 식별 누락 방지 (OSS·드라이버·DICOM 스택 포함)
- 빌드 재현성 증적 (컴파일러·의존성 해시 기록)
- 릴리스 전 잔존 결함의 위험 수용 근거 문서화
- Amendment 2 Legacy 조항 적용 시 gap 분석 명문화

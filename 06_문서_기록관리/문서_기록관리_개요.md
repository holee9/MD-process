# 문서·기록 관리 개요 — v0.2

> **버전**: v0.2 | **최종 갱신**: 2026-05-03 | **작성**: holee9-automation
> v0.1(2026-04-17) 개요 → v0.2 절차흐름도·문서관리대장 양식·X-ray 기술문서 예시 추가

---

## 1. 목적

QMS 운영에 필요한 문서(Document)와 품질기록(Record)을 식별, 승인, 배포, 개정, 보존, 폐기하는 기준을 정한다.

## 2. 적용 범위

- 자사 QMS 문서 전체 (품질매뉴얼, SOP, 작업지침서, 양식)
- 기술문서 (EU MDR Annex II 기술문서, FDA Premarket Submission, MFDS 기술문서)
- 외부문서 (법규·표준·고객사양·공급자 인증서)
- 품질기록 (DHF, DHR, DMR, RMF, 교육기록, 감사기록 등)

## 3. 관련 규제 근거

| 구분 | 국내 MFDS GMP | ISO 13485 | FDA QMSR | EU MDR |
|------|---------------|-----------|----------|--------|
| 문서통제 | 4.2.4 | 4.2.4 | 참조편입 | Annex IX §2 |
| 품질기록 | 4.2.5 | 4.2.5 | 참조편입 | Annex IX §2 |
| 기술문서 | 7.3.10(설계이력) | 7.3.10 | 참조편입 | Annex II, III |
| DMR/DHF/DHR | - | 7.3/7.5.1 | 참조편입 | Annex II 기술문서 |
| 전자기록 | - | - | 21 CFR Part 11 | EU Annex 11 |

## 4. 문서 계층

| 레벨 | 유형 | 예시 | 승인권자 |
|------|------|------|----------|
| Level 1 | 품질매뉴얼 | QM-001 | 대표이사 |
| Level 2 | 절차서(SOP) | SOP-CC-001, SOP-PMS-001 | 부서장 + QA장 |
| Level 3 | 작업지침서·기준서 | WI-CAL-001 교정 지침 | 부서장 |
| Level 4 | 양식·기록 | F-PMS-001, F-CC-001 | 해당 SOP 승인자 |

## 5. 문서관리 절차 흐름

```
┌──────────────────────────────────────────────────────────────┐
│                 문서관리 프로세스 흐름도                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [1단계: 작성 요청]                                          │
│   입력: 신규 문서 필요성 식별 (규제 변경, CAPA, 신제품 등)    │
│   활동: 문서작성요청서(F-DOC-001) 작성                       │
│   출력: 승인된 작성 요청서                                   │
│   판정: QA장 승인 → 문서번호 부여                            │
│                  ↓                                           │
│  [2단계: 초안 작성]                                          │
│   입력: 관련 규제·표준·기존 문서                              │
│   활동: 담당자가 초안 작성 (표준 템플릿 사용)                 │
│   출력: 초안 문서 (Draft 상태)                               │
│   판정: 자체 검토 완료                                       │
│                  ↓                                           │
│  [3단계: 검토·승인]                                          │
│   입력: 초안 문서                                            │
│   활동: 관련 부서 검토 → 의견 반영 → 승인권자 서명           │
│   출력: 승인된 유효 문서 (Effective 상태)                     │
│   판정: 승인 서명·날짜 기재 완료                              │
│          반려 시 → 2단계 복귀                                │
│                  ↓                                           │
│  [4단계: 배포·교육]                                          │
│   입력: 승인 문서                                            │
│   활동: 유효본 배포 (EDMS 또는 인쇄본 관리대장 등록)          │
│         관련 인원 교육 실시                                   │
│   출력: 배포 기록, 교육 기록                                 │
│   판정: 수령·교육 확인 서명                                  │
│                  ↓                                           │
│  [5단계: 개정]                                               │
│   입력: 변경 필요성 (CAPA, 규제변경, 정기검토)                │
│   활동: 개정사유 기록 → 변경통제(SOP-CC-001) 연계            │
│         개정 이력 번호 부여 (R.0→R.1→R.2…)                   │
│   출력: 개정 문서 (이전판 Obsolete 처리)                     │
│   판정: 승인 후 재배포, 폐지본 식별                          │
│                  ↓                                           │
│  [6단계: 보존·폐기]                                          │
│   입력: 폐지문서, 만료기록                                   │
│   활동: 보존기간 확인 → 보존 → 기간 경과 후 폐기             │
│   출력: 폐기 기록                                            │
│   판정: QA 승인 후 폐기                                      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 6. 통제 요구사항

### 6.1 기본 통제
- 제·개정 승인자 지정, 이력 번호 부여 (R.0, R.1 …)
- 유효본/폐지본 구분 (Obsolete 낙인 또는 EDMS 상태 변경)
- 변경사항 표시 (변경 부분 밑줄 또는 개정 이력표에 기재)

### 6.2 전자문서관리(EDMS)
- 21 CFR Part 11 / EU Annex 11 전자기록·전자서명 요건 충족
- 접근권한 관리 (작성·검토·승인·읽기전용 구분)
- 감사추적(Audit Trail) 활성화 — 변경 이력 자동 기록
- 백업·재해복구 절차 수립

### 6.3 외부문서 관리
- 법규·표준·고객사양 식별 → 최신판 유지
- 외부문서 관리대장에 등록 (문서명, 판번, 입수일, 관리부서)
- 개정 모니터링: RA가 분기 1회 규제·표준 개정 확인

| 기록 유형 | 최소 보존 | 근거 |
|-----------|-----------|------|

| 설계이력(DHF) | 제품 수명 + 2년 | ISO 13485 7.3.10 |
| UEF | 제품 수명 + 2년 | IEC 62366-1 |

| 서비스·교정 기록 | 차기 교정 주기 + 1년 이상 | GMP |

## 8. 핵심 기록 체계

### 8.1 설계·제조 파일 체계

| 파일 유형 | 내용 | 자사 X-ray 적용 |
|-----------|------|-----------------|
| DHF (Design History File) | 설계 입력·출력·검증·확인 이력 | X-ray 시스템 각 모델별 DHF |
| DMR (Device Master Record) | 완제품 사양·BOM·도면·SOP 참조 | 시스템·디텍터·SW별 DMR |
| DHR (Device History Record) | 개별 기기 제조 이력 | 시리얼번호별 조립·검사·교정 기록 |
| RMF (Risk Management File) | 위험분석·평가·통제·잔여위험 | ISO 14971 프로세스 산출물 |
| UEF (Usability Engineering File) | 사용적합성 산출물 | IEC 62366-1 §5 산출물 |

### 8.2 EU MDR 기술문서 구조 (Annex II 매핑)

| Annex II 섹션 | 내용 | 자사 문서 매핑 |
|---------------|------|---------------|
| §1 제품 설명·사양 | UDI, 모델, 구성, 액세서리 | DMR + UDI 관리(SOP-UDI-001) |
| §2 제조자 정보 | 설계·제조 사이트 | 품질매뉴얼 §사업장 |
| §3 설계·제조 정보 | 설계이력, 공정 밸리데이션 | DHF + 04_제조/공정밸리데이션 |
| §4 GSPR 적합성 | 일반안전성능요구 매핑 | 01_법규/GSPR_체크리스트 |
| §5 Benefit-Risk 분석 | 위험관리, 임상평가 | RMF + 임상평가보고서 |
| §6 제품 검증·확인 | 시험보고서, SW 밸리데이션 | 05_검사 문서 + SW V&V |
| §6.2(c) 사용적합성 | UEF | §9 UEF 산출물 목록 참조 |

## 9. 사용적합성 엔지니어링 파일(UEF) 기록보관 요건

### 9.1 근거
- IEC 62366-1:2015 + Amd1:2020 §5 — UEF를 QMS 기록체계에 편입
- FDA HFE/UE Guidance (2016) — Premarket Submission에 UEF 핵심 산출물 포함
- EU MDR Annex II §6.2(c) — 사용적합성 평가 결과를 기술문서에 수록

### 9.2 UEF 구성 산출물

| 산출물 | 설명 | 참조 조항 |
|--------|------|-----------|
| Use Specification | 의도된 사용자·환경·환자군·의학적 목적 | IEC 62366-1 §5.1 |
| 사용 관련 위험 분석 | ISO 14971 RMF 연계, HRUS | §5.3–5.4 |
| UI 사양 | 사용자 인터페이스 설계 명세 | §5.5 |
| Formative 평가 기록 | 프로토타입·시뮬레이션 단계별 평가 | §5.7 |
| Summative 평가 계획·보고서 | Critical Task, 합격 기준, 결과 | §5.8–5.9 |
| 잔여 사용 위험 평가 | 수용 불가 잔여 위험 근거·조치 | §5.10 |
| AI 인터랙션 검증 | Automation Bias 시나리오, 투명성 | 자체 기준 + EU AI Act Art.13 |

### 9.3 보관·추적 요건
- UEF는 DHF 내 독립 섹션 또는 독립 파일로 관리
- 기술문서(EU MDR Annex II) 또는 Premarket Submission 매핑 표에 산출물별 위치 명기
- 보존 기간: §7 일반 품질기록 보존 규정과 동일 (제품 수명 + 최소 2년)
- 전자 관리 시 Part 11 / EU Annex 11 요건 충족
- UI 변경·사용자 그룹 추가 시 설계 변경 이력과 연동 갱신

## 12. X-ray 시스템 기술문서 관리 예시

### 12.1 DHF 구성 (일반촬영 X-ray 시스템)

| DHF 섹션 | 포함 문서 | 비고 |
|----------|-----------|------|
| 설계 입력 | 사용자 요구사양, 제품 사양서, 규제 요건 매핑 | GSPR 체크리스트 연계 |
| 설계 출력 | 회로도, 기구도면, SW 설계문서, BOM | SW: IEC 62304 산출물 |
| 설계 검증 | EMC 시험, 전기안전 시험, 방사선 성능시험 | IEC 60601-1, -2-54 |
| 설계 확인 | 임상평가, 사용적합성 평가(UEF) | IEC 62366-1 |
| 설계 이전 | 제조공정 밸리데이션, 교정 절차 | 04_제조/공정밸리데이션 |
| 위험관리 | RMF (FTA, FMEA, 위험평가 매트릭스) | ISO 14971 |
| 사이버보안 | 위협 모델링, Pen-test 결과, SBOM | IEC 81001-5-1 |

### 12.2 문서번호 체계 (권장)

```
[유형]-[카테고리]-[일련번호]  예: SOP-PMS-001, F-CC-003, WI-CAL-002
유형: QM(매뉴얼), SOP(절차서), WI(작업지침), F(양식), PLN(계획서), RPT(보고서)
카테고리: DOC, QMS, DD(설계), MFG(제조), INS(검사), RM(위험), PMS, SUP(공급), TRN(교육)
```

# 교차검증 보고서 — IEC 62304 ↔ ISO 14971 연계성 (2026-04-20)

## 1. 검증 범위
의료용 X-ray 시스템·detector·SW 개발에 대해 IEC 62304 SW 수명주기와 ISO 14971 위험관리 프로세스의 상호 연계·적용 정합성 검증.

## 2. 조항 대응표
| IEC 62304 | ISO 14971:2019 | ISO/TR 24971:2020 | 연계 확인 |
|-----------|----------------|-------------------|-----------|
| §4.3 SW Safety Classification | §5.3 Safety characteristics + §5.5 Risk estimation | Annex A | 일치 — SW 클래스 결정의 입력 = 위험분석 출력 |
| §5.1 개발계획 중 위험관리 | §4.4 RM plan | §4 | 일치 — SW 개발계획과 위험관리 계획 연계 요구 |
| §5.2.2 SW 요구 중 위험통제 | §7.1 Risk control | §7 | 일치 — 위험통제 요구가 SRS에 반영되어야 |
| §5.3 아키텍처 설계 (Class B/C) | §7.1/7.2 Implementation | §7 | 일치 — SOUP 격리, 장애 격리 = 위험통제 조치 |
| §7.1 SW 기여 위해요인 | §5.4 Hazard ID | §5 | 일치 — "SW can contribute to hazardous situations" 명시 |
| §7.2 위험통제 조치 | §7 전반 | §7 | 일치 |
| §7.3 통제 검증 | §7.2 Implementation + §7.3 Verification | §7 | 일치 |
| §7.4 SW 변경·새 위해 식별 | §10.3 / §10.4 | §9 | 일치 — 변경은 RMF 갱신 트리거 |
| §8 구성관리 | (QMS 간접) | — | ISO 13485 §4.2, §7.5.1 과 간접 연계 |
| §9 문제해결 | §10.2 정보 수집 | §8 | 일치 — 불만·결함이 시판후 정보 수집 대상 |

**결론: 조항 수준에서 불일치·공백 없음.** IEC 62304 §7 은 ISO 14971 전체 프로세스의 SW 특화 적용으로 명확히 매핑됨.

## 3. 실무 적용 정합성

### 3.1 안전성 클래스 결정 근거
- 결정 입력: 위험관리 파일의 Hazard → Hazardous Situation → Harm 추정값 (P, S)
- 출력: SW 항목·서브시스템별 Class A/B/C 할당
- **공백 위험**: SW 항목 분할(partitioning)으로 클래스 하향 시 격리 증빙이 아키텍처 문서에 미반영될 가능성 → 아키텍처 설계 체크리스트에 "격리 경계와 그 검증방법" 필수 항목 추가 필요

### 3.2 SOUP 관리와 위험관리
- IEC 62304 §5.3.4, §7.1.3 — SOUP 공표결함(known anomalies) 평가
- ISO 14971 §5.4 — SOUP 결함을 위해요인 식별에 포함 필요
- **불일치**: 다수 프로젝트에서 SOUP 결함을 "SW 결함"으로만 추적하고 ISO 14971 Hazard Traceability Matrix 에 연결하지 않는 사례 존재 → 본 프로젝트는 SOUP 평가 결과를 RMF로 자동 피드하는 양식 필요

### 3.3 시판후 정보와 SW 유지보수
- IEC 62304 §6 유지보수 프로세스의 문제 분석(§6.2.2) = ISO 14971 §10.4 조치 절차와 동기
- **개선**: 문제해결(§9) 로그와 Vigilance/MDR/KAERS 보고를 단일 문제관리 시스템에서 분기 처리 가능하도록 항목 코드 체계 합의 필요

### 3.4 사이버보안 위험의 안전 위험 변환
- IEC 81001-5-1 §4.3 — Security risk 중 환자 안전 영향이 있는 것은 ISO 14971 RMF로 이관
- **공백**: 보안 위험 전용 레지스터와 안전 RMF 간 수동 동기화가 필요 → 본 프로젝트는 "보안→안전 변환 트리거 표" 별도 작성 필요 (AAMI TIR57 Annex C 기반)

### 3.5 AI/ML 모델 성능 드리프트
- IEC 62304 표준 본문은 ML 특화 조항 없음 → AMD 2:2020 + IMDRF N41 + AI Act Art.15 (사후 모니터링) 로 보강
- **불일치 없음**, 단 **공백**: 성능 드리프트 감지 기준치 설정 방법이 표준에 없어 조직 수용 기준 필요 (ISO/IEC 23894 참조)

## 4. 도출 조치
1. SW 아키텍처 설계 체크리스트에 "격리 경계와 검증방법" 항목 추가 → 003/설계개발 SOP 후속 개정
2. SOUP 평가 → RMF Hazard Matrix 자동 연동 양식 초안 작성 → issue-drafts 등록
3. 보안→안전 변환 트리거 표 초안 작성 (AAMI TIR57 Annex C 기반) → 07 위험관리 보강
4. AI/ML 성능 드리프트 수용 기준 정책 초안 → 07 위험관리 + 08 PMS 공동
5. 05 카테고리에 SW 검증 프로토콜 가이드 (IEC 62304 §5.6–5.7 운용판) 신규 작성 → 차기 일자

## 5. 잔여 불일치·개선 필요 사항
- IEC 62304 와 ISO 14971 둘 다 "위험 수용 기준" 자체는 조직 정책에 위임 — 품질매뉴얼 상위 정책 문서에서 확정 필요
- EN 62304/A1 이 OJEU 등재 아니므로 EU MDR GSPR 적합성 증빙 시 state-of-the-art 선언 + gap 분석 필수

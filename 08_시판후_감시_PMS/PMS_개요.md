# 시판후 감시 (Post-Market Surveillance) — v0.2

> **버전**: v0.2 | **최종 갱신**: 2026-05-03 | **작성**: holee9-automation
> v0.1(2026-04-17) 개요 작성 → v0.2 절차흐름·양식·X-ray 적용예시·PSUR 구조 추가

---

## 1. 목적

제조·시판된 의료기기(X-ray 시스템·디텍터·SW)에 대해 체계적으로 시판후 데이터를 수집·분석하고, 안전성·성능의 지속 적합성을 검증하며, 필요 시 시정·예방조치(CAPA) 및 규제 당국 보고를 수행하는 절차를 정한다.

## 2. 적용 범위

- 자사 제조 의료용 X-ray 시스템(일반촬영·투시), 디지털 디텍터(FPD), 영상처리 SW(AI 포함)
- Class IIa(EU MDR 기준) / 2등급(MFDS) / Class II(FDA)

## 3. 관련 규제·표준

| 규제/표준 | 조항 | 요구사항 |
|-----------|------|----------|
| EU MDR 2017/745 | Art. 83–86, Annex III | PMS 계획·PMS 보고서·PSUR·PMCF |
| MDCG 2022-21 | 전문 | PSUR 템플릿·구조 |
| MDCG 2025-10 | 전문 | PMS 실행 가이드(최신) |
| FDA 21 CFR 803 | 전문 | MDR(Medical Device Reporting) 의무 |
| FDA QMSR (21 CFR 820) | ISO 13485 §8.2.1–8.2.2 참조편입 | 피드백·불만처리 |
| 의료기기법 | 제31조, 시행규칙 | 부작용 보고 의무 |
| ISO 13485:2016 | 8.2.1, 8.2.2, 8.2.3, 8.5 | 피드백·불만·CAPA |
| IEC 62304:2006+A1 | §9 | SW 유지보수 |
| IEC 81001-5-1:2021 | §8 | 보안 사고 대응 |
| EU AI Act | Art. 72 | 심각 사고 보고 |

│                                                                 │

│                  ↓                                              │
│  [2단계: 데이터 수집]                                           │
│   입력: 시장 피드백, 규제 DB, 학술문헌, 서비스기록               │
│   활동: 정기/수시 데이터 수집 (아래 §5 참조)                     │

│                  ↓                                              │
│  [3단계: 데이터 분석·경향 평가]                                  │
│   입력: 수집 데이터, 이전 분석 결과                              │
│   활동: 통계 분석, 경향 분석, Benefit-Risk 재평가                │
│   출력: 분석 보고서, 경향 그래프                                 │
│   판정: 수용 가능한 Benefit-Risk 비율 유지 여부                  │

│  [4a: 정기 보고]              [4b: 시정조치 개시]                │

│   또는 PSUR(Class IIa+)        CAPA 개시 (SOP-CAPA 연계)        │
│   PMCF 결과 통합               규제 당국 보고 (§6)              │
│                  ↓                      ↓                       │
│  [5단계: 경영검토 입력]                                         │
│   PMS 결과 → Management Review → 개선 의사결정                  │
│   교육 환류 (10_교육_훈련 §8)                                   │

│                                                                 │

## 5. 데이터 원천 및 수집 주기

| 데이터 원천 | 수집 방법 | 주기 | 담당 |
|-------------|-----------|------|------|
| 고객 불만 (SOP-PMS-001) | 불만접수 시스템 | 수시 | QA |
| 서비스·수리 기록 | FSE 보고서, 부품 교체 이력 | 월 1회 집계 | 서비스팀 |
| 규제 DB (MAUDE, EUDAMED, 의약품안전나라) | DB 검색 | 월 1회 | RA |
| 학술문헌 | PubMed, Embase 검색 | 분기 1회 | RA/임상 |
| 공급자 피드백 | 공급자 부적합 보고 | 수시 | 구매/QA |
| AI 성능 모니터링 | 성능 대시보드 (SOP-AIGOV-001) | 주 1회 | SW팀 |
| 사이버보안 로그 | SIEM/보안 대시보드 | 상시 | 보안팀 |
| PMCF(시판후 임상 추적) | 임상 프로토콜 | 연 1회+ | 임상팀 |

## 6. 보고 기한 (주요국 비교)

| 지역 | 중대 부작용 | 비중대/기타 | 정기 보고 |
|------|-------------|-------------|-----------|
| **EU** | 15일 (사망·심각한 건강 위협은 2일 이내) | 경향 보고 | PSUR — Class IIa: 최소 2년마다 갱신 |
| **US (FDA)** | 30 calendar days (사망·중상은 5 work days) | 말펑션 30일 | 연간(해당 시) |
| **KR (MFDS)** | 7일(사망·중대위해), 15일(중대 부작용) | 기한 없음(내부 기록) | 재평가 보고서(3~5년) |

### 6.1 보고 판정 기준 — X-ray 시스템 맥락

| 사고 유형 | 보고 판정 | 근거 |
|-----------|-----------|------|
| X-ray 튜브 파열 → 환자·사용자 노출 | **즉시 보고** (EU 2일 / KR 7일) | 방사선 과피폭 = 중대 위해 |
| 선량표시 오류(SW 버그) → 과선량 가능성 | 위험평가 후 결정 (CVSS 적용) | AI/SW 오작동 → 잠재 위해 |
| FPD 화소 불량 → 영상품질 저하 | 경향 분석 대상, 빈도에 따라 FSCA 검토 | 진단 영향도 평가 |
| 콜리메이터 오작동 → 조사야 제한 불가 | **즉시 보고** | 방사선 과피폭 위험 |
| AI 진단보조 오진율 상승 | 드리프트 분석 → 임계치 초과 시 FSCA | SOP-AIGOV-001 §드리프트 |
| 사이버보안 침해 → 환자데이터 유출 | EU 72h, 국내 PIPA 통지 | IEC 81001-5-1 §8 |

## 7. PMS 보고서 구조 (PSUR — MDCG 2022-21 기반)

Class IIa 이상 제품에 대해 아래 구조로 PSUR 작성:

| 섹션 | 내용 | 자사 X-ray 적용 |
|------|------|-----------------|
| 1. 제품 식별 | UDI, 모델명, 인증 번호 | X-ray 시스템 각 모델별 기재 |
| 2. PMS Plan 요약 | 수집 방법·주기·분석 기준 | §5 테이블 참조 |
| 3. 수집 데이터 요약 | 불만 건수, 부작용 보고, FSCA 이력 | 유형별(튜브·FPD·SW·AI) 분류 |
| 4. 경향 분석 | 불만 추이, 고장 패턴, 시장 규모 대비 발생률 | 연간 판매대수 대비 불만률 |
| 5. PMCF 결과 | 임상 데이터, 레지스트리 분석 | 영상품질 임상 평가(해당 시) |
| 6. Benefit-Risk 평가 | 편익-위험 비율 재평가 결론 | X-ray 진단 편익 vs 방사선·오진 위험 |
| 7. 조치 및 결론 | 시정조치, 설계변경, 교육 환류 | CAPA 번호, 변경통제(SOP-CC-001) 연계 |

## 8. 사이버보안 사고·AI 부작용 PMS 통합

### 8.1 근거 (추가)
- FDA Postmarket Cybersecurity Guidance (2016, 2023 갱신)
- IEC 81001-5-1:2021 §8 — 보안 사고 대응·교훈 기록
- EU AI Act Art. 72 — 심각한 사고(Serious Incident) 보고 의무
- MFDS 사이버보안 허가심사 가이드 (2025) — 시판후 취약점 관리

### 8.2 사이버보안 사고 PMS 연계

| 사고 유형 | PMS 내 처리 | 보고 대상 |
|-----------|-------------|-----------|
| 취약점 발견 (CVD 접수) | 위험평가 → 패치·완화 → 고객 통보 | CVSS ≥7.0 시 MFDS/FDA 보고 검토 |
| 침해사고 (랜섬웨어, 데이터유출) | FSCA/FSN 발행 검토, 보고 기한 준수 | EU 72h, FDA 즉시/30d |
| 보안 패치 배포 | 변경관리(CC) → PMS 기록 | 중대 변경 시 사전신고 |
| 네트워크 이상 탐지 | 로그 분석 → 위험 재평가 | 내부 기록, 필요 시 보고 |

### 8.3 AI 부작용 PMS 연계

| 사고 유형 | PMS 내 처리 | 보고 대상 |
|-----------|-------------|-----------|
| 오진/과진단(FP/FN) | 성능 모니터링 → 드리프트 분석 → CAPA | 환자 위해 시 부작용 보고 |
| 모델 드리프트 | SOP-AIGOV-001 §드리프트 관리 → PMS 기록 | 성능 저하 ≥임계치 시 FSCA 검토 |
| 편향 발견 (특정 인구집단) | 하위그룹 분석 → 위험평가 갱신 | FDA/MFDS AI 보고 검토 |
| 데이터 품질 이슈 | 데이터셋 재검증 → SOP-AIDATA-001 환류 | 내부 기록 |

### 8.4 통합 모니터링 체계

1. **실시간 탐지**: 보안 SIEM + AI 성능 대시보드 → 이상 알림
2. **주간 검토**: 보안팀·AI팀 합동 주간 이상 리뷰
3. **분기 집계**: PMS 보고서에 사이버·AI 섹션 포함 → 10_교육훈련 환류(§8)
4. **연간 PSUR/PMS 보고서**: 사이버·AI 경향 분석 독립 섹션 수록
5. **경영검토 입력**: 사이버·AI 위험 동향, 패치 현황, 드리프트 지표

## 10. X-ray 시스템 PMS 적용 예시

### 10.1 디지털 디텍터(FPD) 화소 불량 경향 관리

| 단계 | 활동 | 판정 기준 |
|------|------|-----------|
| 수집 | FSE 보고서에서 Dead Pixel 건수 집계 | 월별 집계 |
| 분석 | 모델별·로트별 Dead Pixel 발생률 추이 | 제조 사양 대비 1.5배 이상 → 경고 |
| 조치 | 경고 초과 시 CAPA 개시, 공급자(FPD 패널) 통보 | CAPA 필요 여부 결정 |
| 보고 | PSUR §3에 경향 데이터 수록 | 환자 진단 영향 평가 포함 |

### 10.2 X-ray 튜브 수명 관리 및 사전 안전 조치

- 누적 mAs/조사 횟수 모니터링 → 튜브 교체 권고 임계치 설정
- 비정상 방전·아크 발생 시 즉시 사용 중지 → 불만 접수 → F-PMS-001 평가
- PSUR에 튜브 관련 사고 경향 및 잔여 수명 분석 포함

### 10.3 AI 진단보조 SW 시판후 성능 모니터링

- 주간 AUC/Sensitivity/Specificity 추적 (SOP-AIGOV-001 연계)
- 하위그룹(연령·체형·촬영 부위) 성능 편차 분석
- 드리프트 임계치(AUC 저하 ≥5%p) 초과 시 PMS 기록 + CAPA 검토

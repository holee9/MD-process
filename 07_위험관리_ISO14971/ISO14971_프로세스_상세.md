# ISO 14971:2019 위험관리 프로세스 상세 (+ ISO/TR 24971:2020)

## 1. 목적 및 범위
본 문서는 `위험관리_개요.md` 의 상위 구조를 실행 단위까지 분해한다. 의료용 X-ray 시스템/detector/SW 전 수명주기에 적용.

## 2. 프로세스 단계별 산출물·판정기준

### 2.1 위험관리 계획 (Clause 4.4)
필수 요소:
- 적용 범위(제품·구성·수명주기 단계)
- 역할·책임(R&D, RA, QA, PMS, PRRC 연계)
- 위험 수용 기준 (조직 정책 기반, 객관적 근거 필요)
- 검증 활동 및 결과 편익-위험 분석 방법
- 시판후 정보 수집·검토 주기

X-ray 시스템 특화 고려: 방사선량 한계, 환자 접촉부 생체적합성, 고전압·전자파(EMC), 영상 신뢰도, 사이버보안, AI/ML 성능 드리프트.

### 2.2 위험 분석 (Clause 5)
1. Intended Use / Reasonably Foreseeable Misuse 기술 (5.2)
2. 안전 관련 특성 식별 (5.3) — ISO/TR 24971 Annex A 체크리스트 활용
3. 위해요인 식별 (5.4) — Hazard → Hazardous Situation → Harm 연쇄
4. 시퀀스/확률 추정 (5.5) — P1(Hazardous Situation 발생), P2(Harm 발생) 분해

### 2.3 위험 평가 (Clause 6)
평가 매트릭스 구성: 가능성(P=P1×P2) × 심각성(S). 수용 기준은 본 조직 정책 매트릭스에 따름 (5단계 권장: Negligible / Minor / Serious / Critical / Catastrophic).

### 2.4 위험 통제 (Clause 7)
우선순위:
1. 본질안전 설계 (Inherent safety by design)
2. 보호수단 (방사선 차폐, 인터록, 하드웨어 리미터, 이중화)
3. 정보 제공 (경고·IFU·교육)

각 조치 후 잔여위험 재평가 필수. 통제 자체가 새로운 위해를 유발하지 않는지 확인(7.6).

### 2.5 잔여위험 평가 및 편익-위험 (Clause 8)
모든 통제 후 잔여위험이 수용 기준을 벗어나면 편익-위험 분석으로 정당화. 임상평가 결과·유사 기기 데이터·의학문헌 근거 필요.

### 2.6 위험관리 검토 (Clause 9)
릴리스 전 위험관리 계획 완료 여부 확인, 위험관리 파일 완전성 검증, 편익-위험 수용성 최종 판단, 시판후 정보 수집 계획 운영 준비 확인.

### 2.7 생산·시판후 활동 (Clause 10)
- 10.1 일반: 시판후 정보 수집·검토·조치 체계 운영
- 10.2 정보 수집: PMS, PMCF, 불만, Vigilance, 공급자 품질, 문헌, 규제 안전 경보
- 10.3 정보 검토: 이전 추정과 비교, 새 위해요인 여부
- 10.4 조치: 위험 재평가, 위험통제 조치 변경, 기술문서·IFU 개정

## 3. 수용 기준 설정 가이드 (ISO/TR 24971 §5)
기준 수립 시 고려: 적용 가능 규제, 관련 국제 표준, 일반인 위험 인식, 환자·사용자 기대치, State of the Art. 기준은 정량(확률·심각성 라벨) 또는 정성 허용. 편익-위험 분석에서 "As Low As Reasonably Practicable(ALARP)" 이라는 표현은 ISO 14971:2019에서 삭제됨 — 조직 고유 기준 필요.

## 4. RMF(위험관리 파일) 구성
필수 문서:
1. 위험관리 계획서
2. 위해요인·위험 목록 (Hazard Traceability Matrix)
3. 위험 분석 워크시트 (FMEA·FTA 등)
4. 위험 통제 조치 내역·검증 기록
5. 잔여위험 평가 및 편익-위험 분석
6. 위험관리 검토 보고서 (릴리스 전)
7. 생산·시판후 정보 검토 로그
8. 위험관리 보고서 (생명주기 종료 시 갱신)

## 5. 다른 표준과의 연계
| 연계 표준 | 접점 |
|-----------|------|
| ISO 13485:2016 §7.1 | 제품실현 기획에 위험기반 접근 |
| IEC 62304 §7 | SW 위험관리 (본 문서 2.2~2.5 적용) |
| IEC 62366-1:2015/A1:2020 | Use Error → ISO 14971 Hazardous Situation |
| IEC 60601-1 §4.2 | Essential Performance 및 Single Fault Safe 요구 |
| IEC 81001-5-1:2021 | 보안 위험 → 안전 위험 변환 |
| IEC/TR 60601-4-5 | Security capabilities for PEMS |
| ISO/IEC 23894:2023 | AI 관련 위험관리 가이던스 |

## 6. 정량 방법론 선택 지침 (ISO/TR 24971 Annex B)
| 방법 | 용도 | 프로젝트 적용 예 |
|------|------|------------------|
| PHA | 개념 단계 위해 식별 | 제품 기획 초기 |
| FMEA (기능·설계·공정) | 단일 고장 영향 분석 | detector 센서 고장, 제조 공정 FMEA |
| FTA | 복합 원인·최종 위해 추적 | 과노출 시나리오, AEC 고장 복합원인 |
| HAZOP | 운영 편차 분석 | 워크플로·사용자 인터랙션 |
| STPA | 시스템 이론 기반 제어 상호작용 | SW·HW 통합 제어 분석(AI/ML 포함) |

## 7. 사이버보안 위험 통합 (IEC 81001-5-1 + AAMI TIR57)
- 보안 위협 → 안전 영향 변환 경로 문서화
- Threat Modeling (STRIDE / ATT&CK for Medical Devices)
- SBOM 기반 취약점 모니터링 및 패치 영향 재평가

## 8. 실무 체크포인트
- 위험 수용 기준을 조직 품질 매뉴얼(또는 품질 정책)에 명문화
- Hazard Traceability Matrix 를 설계입력·검증·시판후 데이터와 쌍방향 연결
- 편익-위험 분석은 임상평가(EU MDR) · Indications for Use(FDA 510(k)/De Novo/PMA) 와 정합
- 시판후 단계에서 P1/P2/S 값이 변하면 RMF 개정 및 NB/FDA/MFDS 변경 신고 여부 검토
- SW 안전성 클래스 변경(IEC 62304 §4.3) 이벤트를 위험 재평가 트리거로 명시

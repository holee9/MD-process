# 문서·기록 관리 개요

## 1. 목적
QMS 운영에 필요한 문서(Document)와 품질기록(Record)을 식별, 승인, 배포, 개정, 보존, 폐기하는 기준을 정한다.

## 2. 관련 규제 근거
| 구분 | 국내 MFDS GMP | ISO 13485 | FDA QMSR | EU MDR |
|------|---------------|-----------|----------|--------|
| 문서통제 | 4.2.4 | 4.2.4 | 참조편입 | Annex IX §2 |
| 품질기록 | 4.2.5 | 4.2.5 | 참조편입 | Annex IX §2 |
| 기술문서 | 7.3.10(설계이력) | 7.3.10 | 참조편입 | Annex II, III |
| DMR/DHF/DHR 개념 | - | 7.3(DMR), 7.3.10(DHF), 7.5.1(DHR) | 참조편입(ISO 13485 체계) | Annex II 기술문서 개념 |

## 3. 문서 계층
1. 품질매뉴얼(Level 1)
2. 절차서(SOP, Level 2)
3. 작업지침서·기준서(Level 3)
4. 양식·기록(Level 4)

## 4. 통제 요구
- 제·개정 승인자 지정, 이력 번호 부여(R.0, R.1 …)
- 유효본/폐지본 구분(Obsolete 낙인)
- 전자문서관리시스템(EDMS) 사용 시 Part 11 / EU Annex 11 전자기록·전자서명 요건 충족
- 외부문서(법규·표준·고객사양)도 식별·최신판 유지 관리

## 5. 보존 기간
| 기록 유형 | 최소 보존 |
|-----------|-----------|
| 일반 품질기록 | 제품 수명 + 최소 2년 (ISO 13485 4.2.5) |
| 이식형·장기 사용 | 15년 이상 (MDR Art. 10(8)) |
| 교육기록 | 퇴직 후 일정 기간 별도 관리 |
| 부작용/리콜 기록 | FDA·MDR 요구 기간(일반 5년 이상) |

## 6. 핵심 기록
- DMF / DHF / DHR / DMR (설계이력·설계파일·기기이력·기기마스터)
- 위험관리파일(RMF, ISO 14971)
- 시판후 감시 기록(PMS Plan, PSUR, PMCF)
- 경영검토·내부감사·공급자감사 보고서 (※ FDA 열람 대상)

## 7. UDI 관련 기록
- 21 CFR 830 (FDA) 및 EU MDR Art. 27의 UDI-DI / UDI-PI 기록
- EUDAMED 등록 정보와 내부 DHR 연계

## 8. 정합성 포인트
- FDA QMSR § 820.35(a) Records 조항은 ISO 13485 4.2.5에 추가 요구 덧붙임 → 샘플·서명·날짜 정확성 강화
- EU MDR은 기술문서 자체를 Annex II, III로 직접 구조화 → ISO 13485 DHF를 이 구조로 매핑 관리 필요

## 9. 사용적합성 엔지니어링 파일(UEF) 기록보관 요건

### 9.1 근거
- IEC 62366-1:2015 + Amd1:2020 §5 — Usability Engineering File(UEF)을 QMS 기록체계에 편입
- FDA HFE/UE Guidance (2016) — Premarket Submission에 UEF 핵심 산출물 포함 요구
- EU MDR Annex II §6.2(c) — 사용적합성 평가 결과를 기술문서에 수록

### 9.2 UEF 구성 산출물
| 산출물 | 설명 | 참조 조항 |
|--------|------|-----------|
| Use Specification | 의도된 사용자·환경·환자군·의학적 목적 | IEC 62366-1 §5.1 |
| 사용 관련 위험 분석 | ISO 14971 RMF 연계, HRUS(Hazard-Related Use Scenarios) | §5.3–5.4 |
| UI 사양 | 사용자 인터페이스 설계 명세 | §5.5 |
| Formative 평가 기록 | 프로토타입·시뮬레이션 단계별 평가 결과 | §5.7 |
| Summative 평가 계획·보고서 | Critical Task 정의, 평가 프로토콜, 합격 기준, 결과 | §5.8–5.9 |
| 잔여 사용 위험 평가 | 수용 불가 잔여 위험의 근거 및 조치 | §5.10 |
| AI 인터랙션 검증 (해당 시) | Automation Bias 시나리오, 투명성(EU AI Act Art.13) 가시성 | 자체 기준 |

### 9.3 보관·추적 요건
- UEF는 DHF(Design History File) 내 독립 섹션 또는 독립 파일로 관리
- 기술문서(EU MDR Annex II) 또는 Premarket Submission(FDA) 매핑 표에 UEF 산출물별 위치 명기
- 보존 기간: 일반 품질기록 보존 규정(§5 참조)과 동일하게 제품 수명 + 최소 2년 적용
- 전자 관리 시 Part 11 / EU Annex 11 전자기록 요건 충족 필수
- 변경 시(UI 변경, 사용자 그룹 추가 등) UEF를 해당 설계 변경 이력과 연동 갱신

### 9.4 정합성 포인트
- 05_검사_시험_밸리데이션의 UE 계획서(IEC 62366-1)와 본 기록보관 규정이 산출물 목록 기준으로 일치해야 함
- 07_위험관리_ISO14971의 RMF와 HRUS 교차 참조 보장

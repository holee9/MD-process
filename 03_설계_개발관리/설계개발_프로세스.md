# 설계·개발 관리 프로세스

## 1. 근거
- ISO 13485 7.3
- MFDS GMP 별표2 7.3
- 21 CFR 820 (QMSR - ISO 13485 편입)
- EU MDR Annex II (기술문서)

## 2. 단계별 요구사항
| 단계 | 요구사항 | 산출물 |
|------|----------|--------|
| 기획 | 설계·개발 계획 수립 | 설계개발 계획서 |
| 입력 | 사용자·규제·성능·안전 요구사항 정의 | 설계입력서(DI) |
| 출력 | 입력요구 대응, 검증 가능 | 도면·사양서·S/W 명세 |
| 검토 | 단계별 설계검토 | 설계검토 회의록 |
| 검증(V) | 출력이 입력을 충족하는지 확인 | 검증 프로토콜·보고서 |
| 밸리데이션(V) | 실사용 조건에서 의도된 사용 충족 | 임상평가·사용성평가 |
| 이관 | 생산으로의 설계 이관 | 제조 이관 기록 |
| 변경 | 변경통제 | 변경요청서·영향평가 |

## 3. 설계이력파일(DHF) / 위험관리 연계
- DHF에 위험관리 파일(RMF) 참조 포함
- 설계입력에 위험통제 요구사항 반영

## 4. 관련 표준
- IEC 62366-1 (사용성 엔지니어링)
- IEC 62304 (의료기기 소프트웨어 수명주기)
- IEC 60601 시리즈 (전기의료기기 안전성)

## 5. 실무 체크포인트
- 설계입력에 규제요구·사용자요구·성능요구·안전요구를 모두 포함
- 각 단계 이동 전 독립 검토자 승인
- 변경 시 위험관리 재평가 트리거 정의

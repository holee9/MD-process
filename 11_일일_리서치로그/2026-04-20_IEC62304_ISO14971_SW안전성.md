## 주요 확인 사항

### 1) IEC 62304 표준 현행 상태
- 현행: IEC 62304:2006 + A1:2015 + A2:2020 (컨솔 미발행, 3개 문서 조합)
- EN 62304:2006/A1:2015 는 OJEU 미등재 → MDCG 2021-5 "state-of-the-art" 적용 근거로만 인정
- A2:2020 핵심: Legacy Software 조항 신설, 사이버보안 참조 강화(IEC 81001-5-1 연계)
- FDA 인정표준 DB 부분 등재 (2023 Premarket Cybersecurity Guidance와 병행)

### 2) SW 안전성 클래스 의사결정
- A/B/C 결정 = 위험통제 기여도 × 고장 시 심각성 (ISO 14971 위험분석 결과에 근거)
- Amendment 1:2015 — 외부 수단(HW 인터록 등)으로 위험통제가 완전 이전되면 클래스 하향 가능
- X-ray 제품: AEC·노출 제어·차폐 인터록 SW = Class C (사망·중대상해 시나리오)

### 3) ISO 14971:2019 ↔ 이전판 변경점 재확인
- "ALARP" 표현 삭제 → 조직 고유 수용 기준 수립 필요
- 편익-위험 분석 범위 확대 — 잔여위험이 수용 기준을 초과할 때의 정당화 절차 강화
- 시판후 정보 검토(§10.3)에 "이전 추정과의 비교" 명시
- ISO/TR 24971:2020 Annex B가 PHA·FMEA·FTA·HAZOP·STPA 방법론 가이드 제공

### 4) SW와 위험관리 프로세스 접점 (IEC 62304 §7 ↔ ISO 14971)
| IEC 62304 §7 | ISO 14971 대응 |
|--------------|----------------|
| 7.1 SW 기여 위해요인 분석 | §5.4 Hazard identification |
| 7.2 위험통제 조치 | §7 Risk control |
| 7.3 통제 조치 검증 | §7.2 Implementation + §7.3 Verification |
| 7.4 SW 변경 시 위험 재평가 | §10.3/10.4 Review & Action |

### 5) AI/ML SW 특화 포인트
- IEC 62304 수명주기에 AI Act Art. 9/10/11/12/13 + IMDRF N41 + FDA GMLP 오버레이
- PCCP (사전 변경관리 계획) 는 FDA·MDR·AI Act Art. 43(4) 3자 모두 유사 개념 존재 → 단일 문서로 정합 가능
- 모델 성능 드리프트 모니터링을 PMS 계획에 포함
